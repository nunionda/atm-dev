# 시니어 멘토의 팁: "좋은 PR"을 위한 가이드라인

    1. 단순히 목적을 아는 것을 넘어, 팀에 기여하는 PR을 만들려면 다음 제약 사항을 고려하세요.

    2. Small Size: 하나의 PR은 하나의 기능(또는 버그 수정)만 담아야 합니다. 너무 큰 PR은 리뷰어의 집중력을 떨어뜨려 버그를 놓치게 만듭니다.

    3. Context 제공: <task>, <context>를 명시하듯, 왜 이 작업을 했고 어떤 부분을 중점적으로 봐달라는 가이드를 본문에 포함하세요.

    4. Self-Review: PR을 올리기 전, 본인이 먼저 코드를 한 줄씩 읽으며 주석이나 불필요한 로그를 정리했는지 확인하세요.
