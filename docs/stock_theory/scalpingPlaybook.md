## 2. 핵심 프레임워크: AMT 3단계 필터

파비오의 모든 의사결정은 Auction Market Theory(AMT)에 기반한 3단계 필터를 통과해야 한다. **하나라도 빠지면 관망.**

| # | 단계 | 설명 |
|---|------|------|
| 1 | **Market State** | 시장이 Balance(균형)인지 Imbalance(불균형)인지 판단. 균형 상태에서는 70% 시간 가격이 갇혀 있음. |
| 2 | **Location** | Volume Profile로 핵심 레벨 식별: LVN(저볼륨노드), POC(Point of Control), VAH/VAL |
| 3 | **Aggression** | 풋프린트에서 오더플로우 확인: 큰 프린트, 임밸런스, 강한 CVD 압력. **공격성 없으면 = 거래 없음.** |

## 4. 셋업 모델 1: Trend Continuation (추세 지속)

시장이 불균형 상태에서 방향성 있게 밀어낼 때 사용. **NY 세션에서 가장 효과적.**

| Step | Action | Detail / Rule |
|------|--------|---------------|
| 1 | Market State 확인 | 시장이 불균형 상태인지 확인. 이전 밸류에서의 이탈과 모멘텀이 보여야 함. 회전만 하면 스킵. |
| 2 | 임펄스 레그 VP 적용 | 구조를 깨뜨린 임펄스 레그에 Volume Profile 적용 → LVN(저볼륨 노드) 식별. |
| 3 | LVN에서 대기 | 가격이 LVN으로 되돌아올 때까지 인내심을 가지고 대기. **맹목적 지정가 주문 금지.** |
| 4 | 공격성 확인 | 풋프린트에서 추세 방향의 공격성 확인. 롱: 큰 매수 버블/임밸런스. 숏: 큰 매도 버블. |
| 5 | 진입 실행 | 공격성이 확인된 순간 진입. **No aggression = No trade.** |
| 6 | 손절 배치 | 공격적 프린트 바로 너머 + 스윙 고/저점 앞 1~2틱 버퍼. |
| 7 | BE 이동 | CVD가 강한 압력을 보이면 빠르게 본전으로 이동. |
| 8 | 타겟 청산 | 이전 Balance POC에서 풀 포지션 청산. (70% 확률로 균형에서 반전) |

---

## 5. 셋업 모델 2: Mean Reversion (평균 회귀)

시장이 균형 밖으로 브레이크아웃을 시도했지만 실패할 때 사용. **런던 세션이나 압축된 시장에서 효과적.**

| Step | Action | Detail / Rule |
|------|--------|---------------|
| 1 | Market State 확인 | 시장이 Balance/컨솔리데이션 상태. 전일 프로파일을 균형 기준으로 사용. |
| 2 | 브레이크아웃 실패 관찰 | 가격이 균형 밖으로 밀고 나갔다가 실패하는 것을 확인. **첫 번째 복귀에 진입하지 않음.** |
| 3 | 리클레임 후 풀백 대기 | 명확한 균형 내 리클레임 → 리클레임 레그의 풀백 → LVN에 VP 적용. |
| 4 | 공격성 확인 | 풀백이 LVN에 도달할 때 스냅백 방향의 공격성 확인 후 진입. |
| 5 | 손절 배치 | 공격적 프린트 너머 + 명확한 고/저점 앞 1~2틱 버퍼. **절대 손절 확대 금지.** |
| 6 | 타겟 청산 | Balance POC(밸류 중심)에서 전체 포지션 청산. 범위 반대측으로 연장하지 않음. |

---

## 6. Triple-A 모델 (Absorption → Accumulation → Aggression)

기관 트레이딩의 전체 라이프사이클 반영. **세 단계가 모두 정렬될 때만 최고 확신도 시그널 발생.**

| 단계 | 정의 | 차트에서 보이는 것 |
|------|------|-------------------|
| 1. **Absorption** (흡수) | 대형 패시브 오더가 공격적 흐름을 흡수 | 높은 볼륨 + 작은 가격 이동 (하이볼륨/로우레인지 캔들) |
| 2. **Accumulation** (축적) | 레인지 수축으로 나타나는 포지션 구축 구간 | 줄어드는 볼륨 + 점점 작아지는 캔들 범위 (Tight consolidation) |
| 3. **Aggression** (공격) | 확장 무브에서의 공격적 진입 시점 | 큰 델타 전환 + 스택드 임밸런스 + 볼륨 급증 (한 방향 폭발) |

## 7. 셋업 등급 시스템 (A / B / C)

컨플루언스(조건 중첩도)에 따라 셋업을 등급화하고 리스크를 조절한다.

| 등급 | 컨플루언스 조건 | 리스크 배분 |
|------|----------------|-------------|
| **A** | 구조 + VWAP/AVWAP 레벨 + 임밸런스/흡수 + 모멘텀 전환 | 일일 최대 트레이드당 리스크 |
| **B** | 구조 + 1개 테이프 신호 (임밸런스 OR 델타 전환) | A의 50% 리스크 |
| **C** | 구조만 있고 테이프 보류 | A의 25% 리스크, 빠른 스크래치 |

- **스케일 업:** 세션 순이익이 양(+)일 때만 리스크 티어 상향
- **스케일 다운:** 연속 2패 후 C 리스크로 축소

## 8. 리스크 관리 규칙

| 규칙 | 세부 사항 |
|------|-----------|
| 트레이드당 리스크 | 계좌의 **0.25% ~ 0.5%** |
| 일일 손실 한도 | 하드 맥스 설정, 도달 시 거래 중단 |
| 연속 손절 제한 | **3회 손절 시 당일 거래 중단** |
| 손절 배치 | 공격적 프린트 너머 + 1~2틱 버퍼 |
| 손절 확대 금지 | 틀리면 즉시 틀려야 함. **절대 손절 확대 금지** |
| BE(본전) 이동 | CVD가 강한 압력을 보이면 빠르게 이동 |
| 포지션 관리 | 한 번에 1개 포지션만. 분할 진입 가능하나 동시 복수 금지 |
| 티켓 규칙 | 1아이디어 = 1티켓. 같은 임펄스에서 2번 실패 시 다음으로 넘어감 |
| 드로다운 목표 | **20% 이하 유지** |

## 9. 포지션 빌딩 & 컴파운딩

1. 세션 초반에 작은 리스크로 시작 (0.25%)
2. 초반 이익을 확보하면 그 이익을 다음 포지션에 활용 (리스크 프리 구축)
3. 성과에 따라 리스크 티어를 점진적으로 상향 조정
4. 강한 모멘텀이 있는 날에 최대한 수익 극대화
5. 연속 손실 시 즉시 최소 리스크로 복귀

> **핵심 원칙:** 세션 초반 이익으로 시드 머니를 만들고, 그 시드 머니로 포지션을 키워나간다. 절대로 원금을 위험에 노출하지 않는다.

| 5 | 공격성 확인 | 풋프린트/오더플로우에서 방향성 있는 공격성. **없으면 절대 진입 금지.** |
| 6 | 셋업 등급 판정 | A/B/C 등급에 따라 사이즈 조절 |

| 9 | 포지션 관리 | CVD 강하면 BE 이동. 흡수 실패 시 즉시 스크래치. |
| 10 | 청산 실행 | 타겟(POC)에서 풀 포지션 청산. 욕심 부리지 않기. |
| 11 | 후처리 | 거래 기록, 스크린샷, 복기. 다음 기회 대기 or 세션 종료. |

## 11. DO vs DON'T

### DO (해야 할 것)
- 세션 전 내러티브 설정
- 3단계 필터 모두 충족 후 진입
- 작은 리스크로 시작, 컴파운딩
- 틀리면 즉시 인정하고 컷
- 1아이디어 = 1티켓 규칙 준수
- 모든 거래 기록하고 복기
- 3회 손절 시 당일 중단
- NY 세션 집중 (유동성 최고)

### DON'T (하지 말아야 할 것)
- 내러티브 없이 차트만 보고 진입
- 조건 불완전한 상태에서 진입
- 처음부터 큰 사이즈로 시작
- 손절 확대 또는 희망 트레이딩
- 같은 임펄스에 반복 진입 (복수 트레이딩)
- 거래 후 분석 없이 다음 거래로 이동
- 손실 회복을 위해 과대 거래
- 비활성 시간대에 스캘핑 시도

## 개발 구현 참고사항

### 시그널 엔진 매핑

```
AMT 3단계 필터 → 코드 구현 우선순위:

1. Market State Detection
   - Balance: 가격이 VAH-VAL 범위 내 70%+ 시간 체류
   - Imbalance: 이전 밸류 영역 이탈 + 방향성 모멘텀

2. Location (Volume Profile 기반)
   - POC (Point of Control): 최다 거래량 가격대
   - VAH/VAL (Value Area High/Low): 거래량 70% 집중 구간
   - LVN (Low Volume Node): 저거래량 노드 → 핵심 진입 레벨

3. Aggression Detection
   - Delta 급증: 매수/매도 공격성 측정
   - 임밸런스 비율: 한 방향 압도적 틱 비율
   - CVD 방향성: 누적 델타 추세 확인
```

### 셋업 선택 로직

```
if market_state == IMBALANCE:
    model = TREND_CONTINUATION
    # LVN 풀백 대기 → 공격성 확인 → 진입
    # 타겟: 이전 Balance POC
elif market_state == BALANCE:
    model = MEAN_REVERSION
    # 브레이크아웃 실패 관찰 → 리클레임 확인 → LVN 풀백 → 진입
    # 타겟: Balance POC (밸류 중심)
```

### 리스크 파라미터 상수

```python
RISK_PER_TRADE_MIN = 0.0025   # 0.25%
RISK_PER_TRADE_MAX = 0.005    # 0.5%
DAILY_LOSS_LIMIT = "hard_max"  # 설정값 도달 시 거래 중단
CONSECUTIVE_STOP_LIMIT = 3     # 3회 연속 손절 시 당일 중단
MAX_DRAWDOWN_TARGET = 0.20     # 20%
STOP_BUFFER_TICKS = 2          # 1~2틱 버퍼

# 셋업 등급별 리스크 배수
GRADE_A_MULTIPLIER = 1.0       # 풀 리스크
GRADE_B_MULTIPLIER = 0.5       # A의 50%
GRADE_C_MULTIPLIER = 0.25      # A의 25%

# 스케일링 규칙
SCALE_UP_CONDITION = "session_pnl > 0"
SCALE_DOWN_TRIGGER = 2         # 연속 2패 후 C로 축소
```

### Triple-A 감지 패턴

```
Phase 1 (Absorption):  high_volume AND low_range  → 흡수 감지
Phase 2 (Accumulation): declining_volume AND narrowing_range → 축적 감지
Phase 3 (Aggression):  delta_spike AND stacked_imbalance AND volume_surge → 폭발 감지

모든 Phase 정렬 = 최고 확신도 시그널 (Grade A)
```
