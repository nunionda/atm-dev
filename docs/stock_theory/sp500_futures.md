## 3. 증거금 (Margin Requirements)

### 3.1 CME 거래소 증거금

| 증거금 유형 | ES (E-mini) | MES (Micro) |
|-------------|-------------|-------------|
| **개시 증거금 (Initial)** | ~$15,000~$16,000 | ~$1,500~$1,600 |
| **유지 증거금 (Maintenance)** | ~$13,200~$14,200 | ~$1,320~$1,420 |
| **오버나이트 증거금** | ~$24,200 (일부 브로커) | ~$2,420 |
| **데이트레이드 증거금 (브로커)** | $400~$800 | $40~$80 |

### 3.2 증거금 핵심 사항

- 증거금은 명목가치의 약 **5%** → 약 **20배 레버리지**
- CME는 시장 변동성에 따라 증거금을 수시 조정
- 극심한 변동성 시 증거금이 대폭 인상될 수 있음
- 브로커 데이트레이드 증거금은 당일 청산 포지션에만 적용
- 극단적 변동성 시 브로커가 거래소 유지증거금 100%를 요구할 수 있음

### 3.3 레버리지 예시

```
ES 1계약 매수 (S&P 500 = 5,660):
  명목가치: 5,660 × $50 = $283,000
  개시증거금: ~$15,500 (약 5.5%)
  레버리지: ~18.3배

  10포인트 상승 시 수익: $500 → 증거금 대비 +3.2%
  10포인트 하락 시 손실: -$500 → 증거금 대비 -3.2%
```

## 5. 가격제한폭 / 서킷브레이커 (Circuit Breaker)

### 5.1 야간 세션 (17:00~08:30 CT)

| 제한폭 | 방향 | 효과 |
|--------|------|------|
| **±7%** | 상승/하락 모두 | 제한폭 내에서만 거래 (매매 중단 아님) |

### 5.2 정규장 (08:30~15:00 CT) — NYSE Rule 7.12

| 단계 | 하락폭 | 조치 | 지속시간 |
|------|--------|------|----------|
| **Level 1** | **-7%** (14:25 CT 이전) | 미국 전체 주식+선물 **매매 중단** | 15분 |
| **Level 2** | **-13%** (14:25 CT 이전) | 미국 전체 주식+선물 **매매 중단** | 15분, 이후 20%로 확대 |
| **Level 3** | **-20%** | **당일 거래 종결** | 다음 세션 개장까지 |

### 5.3 후장 (14:25~15:00 CT)

- **20% 제한만 적용** (7%, 13% 중단 미발동)

### 5.4 동적 서킷브레이커 (Dynamic CB)

| 항목 | 내용 |
|------|------|
| **폭** | ±3.5% |
| **조건** | 1시간 내 3.5% 이상 변동 |
| **효과** | **2분간 매매 일시정지** |
| **적용** | 모든 세션 |

### 5.5 KOSPI200 선물과 비교

| 항목 | ES (S&P 500) | KOSPI200 선물 |
|------|-------------|---------------|
| **1단계** | -7% → 15분 중단 | ±8% |
| **2단계** | -13% → 15분 중단 | ±15% |
| **3단계** | -20% → 당일 종결 | ±20% |
| **동적 CB** | 3.5%/1시간 → 2분 정지 | SIDECAR: 5%/1분 → 5분 정지 |

## 9. 롤오버 (Rollover)

### 9.1 분기별 롤오버 구조

1. ES 계약은 분기별로 **결제월 셋째 금요일**에 만기
2. **롤 데이트 (Roll Date)**: 만기 전 월요일 (거래량 이동 시작)
3. 실무적으로 결제월 **둘째 목요일**에 대부분의 거래량이 차근월로 이동
4. 롤 이후 차근월이 "프론트 먼스(근월물)"가 됨

### 9.2 2026년 롤 스케줄

| 이벤트 | 롤 데이트 (거래량 이동) | 만기일 (최종거래일) | 신규 근월물 |
|--------|----------------------|-------------------|------------|
| **H26 → M26** | 2026년 3월 16일 (월) | 2026년 3월 20일 (금) | ESM26 (6월) |
| **M26 → U26** | 2026년 6월 15일 (월) | 2026년 6월 18일 (목) | ESU26 (9월) |
| **U26 → Z26** | 2026년 9월 14일 (월) | 2026년 9월 18일 (금) | ESZ26 (12월) |
| **Z26 → H27** | 2026년 12월 14일 (월) | 2026년 12월 18일 (금) | ESH27 (3월) |

### 9.3 롤오버 실무 팁

- 대부분의 트레이더는 공식 롤 데이트 **2~3 거래일 전**에 롤오버
- 롤 당일에도 만기 계약에 거래량이 남아있으며, 다음 날 완전히 이동
- **트리플 위칭 (Triple Witching)**: 셋째 금요일에 주식옵션 + 지수옵션 + 지수선물 동시 만기 → 극심한 거래량 급증
- **캘린더 스프레드 주문**으로 슬리피지 최소화하며 롤 가능
- 롤오버 시 근월-차월 간 **베이시스 (Basis)** 차이 확인 필수

## 11. 세금 (미국 Section 1256)

미국 세법 Section 1256에 따라 규제 선물 계약은 특별한 세금 혜택을 받습니다:

| 항목 | 내용 |
|------|------|
| **60/40 Rule** | 이익의 60%는 장기 양도소득, 40%는 단기 양도소득으로 과세 |
| **장기 세율** | 최대 15~20% |
| **단기 세율** | 최대 37% |
| **혼합 세율** | 약 **26.8%** (최고세율 기준) |
| **Mark-to-Market** | 연말 미실현 손익도 과세 대상 |
| **손실 이월** | 3년 전 소급 가능 |

> 주식 단기매매 (최대 37%) 대비 선물이 세금 면에서 유리

## 12. ATS 시스템 통합을 위한 핵심 파라미터

현재 `config.yaml`의 `sp500_futures` 섹션과 연계하여 거래 조건 파라미터:

```yaml
# S&P 500 선물 상품 규격
sp500_futures_specs:
  # E-mini (ES)
  es_multiplier: 50            # USD per index point
  es_tick_size: 0.25           # index points
  es_tick_value: 12.50         # USD per tick
  es_notional: 283000          # 대략 명목가치 (S&P ~5,660)

  # Micro E-mini (MES)
  mes_multiplier: 5            # USD per index point
  mes_tick_size: 0.25          # index points
  mes_tick_value: 1.25         # USD per tick
  mes_notional: 28300          # 대략 명목가치

  # 거래 일정
  contract_months: [3, 6, 9, 12]   # H, M, U, Z
  trading_hours: "Sun 17:00 - Fri 16:00 CT"
```

## 13. 트레이딩 시 주요 고려사항

### 13.1 장점

- **극도의 유동성**: 세계 최고 수준, 1틱 스프레드 상시 유지
- **거의 24시간 거래**: 야간 리스크 이벤트에 즉각 대응 가능
- **양방향 매매**: 롱/숏 동일 조건, 공매도 제한 없음
- **세금 혜택**: 60/40 Rule로 단기매매에도 유리한 세율
- **MES로 소액 진입**: $1,500 증거금으로 시작 가능
- **낮은 거래 비용**: 명목가치 대비 0.01% 수준

### 13.2 리스크

- **높은 레버리지**: ~20배 레버리지로 소액 변동에도 큰 손익
- **갭 리스크**: 일일 1시간 점검 외에도 뉴스에 의한 급변동
- **증거금 콜**: 변동성 급등 시 증거금 인상 → 추가 자금 필요
- **롤오버 비용**: 분기마다 포지션 이전 필요, 베이시스 차이 발생
- **시차**: KST 기준 미국 정규장이 심야 (23:30~06:00)

### 13.3 ATS 전략 적용 시 유의사항
